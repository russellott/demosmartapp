# 📁 Provider Directory Browser

A FHIR-compliant web application for exploring publicly accessible Provider Directory APIs. This app allows you to browse practitioners, organizations, locations, healthcare services, and practitioner roles without requiring authentication.

## Features

- 🏥 **Multiple Public Provider Directories** – Select from several FHIR-compliant provider directory endpoints
- 📋 **Search Providers** – Filter by name, specialty, organization, or location
- 🔍 **Debug Console** – View API requests, response times, and network details
- 📊 **Statistics** – See counts of practitioners, organizations, locations, and more
- 🎨 **Modern UI** – Clean, responsive design matching the Patient Access app
- 🌐 **No Authentication Required** – All endpoints are publicly accessible

## Supported Provider Directories

### SMART Health IT Sandbox
- **URL:** `https://launch.smarthealthit.org/v/r4/sim/.../fhir`
- **Description:** Public testing sandbox with sample provider data
- **Resources:** Practitioners, Organizations, Locations, PractitionerRoles

### Logica Health Open Sandbox
- **URL:** `https://api.logicahealth.org/FHIRResearchSandbox/open/fhir`
- **Description:** Advanced testing sandbox with comprehensive provider directory data
- **Resources:** Practitioners, Organizations, Locations, HealthcareServices, PractitionerRoles

### FHIR Test Server (HAPI)
- **URL:** `http://hapi.fhir.org/baseR4`
- **Description:** Public HAPI FHIR test server with provider directory resources
- **Resources:** Practitioners, Organizations, Locations, PractitionerRoles

## How to Use

1. Open `index.html` in your browser
2. Select a provider directory from the available options
3. Click "Browse Directory"
4. Explore providers using the search bar or by scrolling through categories
5. Toggle the debug panel (top right) to see API requests and performance metrics

## Supported FHIR Resource Types

- **Practitioner** – Healthcare professionals (doctors, nurses, etc.)
- **Organization** – Healthcare organizations, clinics, hospitals
- **Location** – Geographic locations where services are provided
- **HealthcareService** – Services offered by organizations
- **PractitionerRole** – Relationships between practitioners and organizations

## FHIR Compliance

This app uses the FHIR R4 (Release 4) standard for querying provider directory data. All endpoints follow the [FHIR Search specification](https://www.hl7.org/fhir/search.html) for resource retrieval.

### Example API Calls

```
# Get all practitioners
GET {baseUrl}/Practitioner

# Get all organizations
GET {baseUrl}/Organization

# Get all locations
GET {baseUrl}/Location

# Search for practitioners by name
GET {baseUrl}/Practitioner?name=smith

# Get all healthcare services
GET {baseUrl}/HealthcareService
```

## Debug Console
